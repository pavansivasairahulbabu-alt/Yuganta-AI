PROJECT HANDOVER DOCUMENTATION

1. Project Overview
- Project Name: YugantaAI
- Purpose: Production EdTech platform for selling and delivering  courses, capturing leads, and managing mentorship workflows for learners.
- Target Users:
  - Students/Learners
  - Instructors
  - Mentors
  - Admin

2. Tech Stack
- Frontend: React 19, Vite, React Router v7, Tailwind CSS
- Backend: Node.js, Express, JWT auth, helmet, cors, compression, express-rate-limit
- Database: MongoDB with Mongoose
-Google OAuth API and Cloudinary API are used
- Hosting: Frontend on Vercel, Backend on Render

4. Project Setup (How to Run)
1) Clone repository
2) Backend:
   - cd backend
   - npm install
   - create .env 
   - set required variables: MONGODB_URI, JWT_SECRET
   - set recommended email variables: BREVO_API_KEY, BREVO_FROM_EMAIL
   - npm run dev
3) Frontend:
   - cd ../frontend
   - npm install
   - create .env with VITE_API_URL (if needed)
   - npm run dev
4) Open app at http://localhost:5173

Additional requirements:
- MongoDB connectivity
- Cloudinary keys for media upload flows
- Google OAuth client for social login

7. Database Structure
- users: fullName, email, password, verification/otp fields, enrolledCourses, assignedInstructor, assignedMentor
- instructors: name, email, password, bio, photo, expertise, approval/status, courses
- mentors: name, expertise, email, password, bio, photo, approval/status, assignedSessions
- courses: title, description, instructor data, category/level/pricing, modules/videos, media links
- blogs: title, slug, excerpt, content, author, category, tags, featured, likes/views
- leads: contact details, course info, preferred mode/time, status/type/source
- mentorshipsessions: user/instructor/mentor links, date/time, notes, meeting link, status lifecycle fields

12. Deployment
Frontend:
- Vercel build: npm run build
- Output: dist
- SPA rewrite to index.html

Backend:
- Render-hosted Node/Express API
- Dockerfile available
- Requires environment variables on hosting platform

Database:
- MongoDB Atlas

Domain:
- Frontend on Vercel domain, backend on Render URL used via VITE_API_URL

Deployment flow:
1) Push to GitHub
2) Deploy backend on Render with env vars
3) Deploy frontend on Vercel with VITE_API_URL set
4) Validate auth, emails, CORS, media, and mentorship flows
